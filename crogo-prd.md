# Crogo 产品需求文档 (PRD)

> 版本: v0.3 · 状态: 待确认 · 2024-12

---

## 1. 产品定位

### 1.1 一句话描述
为医药CRO打造的文档智能平台——批量构建知识库，上传方案自动填充模板，一键生成文档。

### 1.2 目标用户
- **主要用户**：CRO公司的统计师、数据管理员、医学写作者
- **次要用户**：项目管理人员、总经理

### 1.3 解决的核心问题
| 问题 | 现状 | Crogo方案 |
|---|---|---|
| 重复劳动 | 每个项目手动写3份大文档，反复翻PDF抄数据 | 一次建知识库 → 新方案自动匹配填充 |
| 历史知识沉睡 | 做完的项目知识散落在个人电脑里，换人=失传 | 统一知识库，可检索可复用，越用越聪明 |
| 模板差异 | 每个申办方有自己模板体系 | 上传即用，自动解析，规则学习 |
| 跨文档矛盾 | 同一方案，三份文档说法不一致 | 同一知识库驱动，自动保证一致 |

---

## 2. 核心工作流（3步）

```
Step 1: 上传模板
        └─ 拖入 .docx → 自动解析章节/占位符 → 用户确认映射 ✓
           → 更正面签 ✏️ → 系统学习规则 → 下次自动匹配

Step 2: 知识库（核心）
        └─ 从0到1：批量上传所有历史方案PDF → 构建RAG知识库
        └─ 日常：新方案上传 → 自动查询知识库辅助提取
        └─ 生长：生成文档后 → AI提炼可入库知识 → 用户确认 → 入库

Step 3: 文档生成
        └─ 选模板 + 新上传的方案 → 一键生成 → 下载
```

## 3. 功能清单

### 模块A：模板管理（P0）

| ID | 功能 | 描述 |
|---|---|---|
| A01 | 上传模板 | 拖入 .docx 文件，自动上传 |
| A02 | 自动解析结构 | 系统读取模板的章节结构（编号+标题）、占位符（<xxx>）、表格、附件 |
| A03 | 映射确认界面 | 用户逐项确认解析结果：章节编号对✓，占位符识别对✓ |
| A04 | 更正则学习 | 用户修正→系统记录规则→下次自动匹配修正逻辑 |
| A05 | 模板标签 | 给模板打标签（如"恒瑞版""DMC"），方便筛选 |
| A06 | 模板版本 | 同一模板支持多版本 |
| A07 | 模板列表 | 已有模板一览，搜索/筛选 |

### 模块B：知识库（P0 · 核心差异化）

| ID | 功能 | 描述 |
|---|---|---|
| B01 | **批量入库** | 用户一次性拖入 n 份历史方案PDF（或各种格式），系统批量构建知识库。底层用RAG/向量数据库/或其他高效方案（开源优先，低token消耗） |
| B02 | **智能查询** | 新方案上传后，自动查询知识库获取参考信息，辅助字段提取 |
| B03 | **知识提炼（反馈闭环）** | 文档生成完后，AI自动分析新方案有哪些值得入库的知识 → 推送给用户确认 → 确认后入库。知识库持续生长 |
| B04 | 单方案上传+提取 | 上传单份PDF，AI提取结构化字段 |
| B05 | 校验界面 | 逐字段展示提取结果，用户确认✓或修改✏️ |
| B06 | 知识检索 | 全文搜索历史知识库（"搜胰腺癌终点定义"） |
| B07 | 知识库概览 | 展示知识库规模（入库项目数、文档数、知识条目数） |

**知识库生长模型：**
```
第1天: 用户批量丢入 50 个历史项目PDF → 知识库建立
    ↓
第2天: 新项目「胰腺癌RWS」→ 上传 → 查询知识库辅助提取 → 生成文档
    ↓
第3天: 生成完毕 → AI分析 → "发现2条新知识：伊立替康脂质体的安全监测模式"
        用户确认 ✓ → 知识库更新
    ↓
第90天: 知识库已积累 200+ 项目 → 新项目提取准确率 95%+
        客户越用越离不开（切换成本极高）
```

**与通用RAG/GPT的区别：**
- 通用GPT：临时问答，不积累
- Crogo知识库：**项目做完，知识留下。系统越用越聪明，形成数据飞轮**

### 模块C：文档生成（P0）

| ID | 功能 | 描述 |
|---|---|---|
| C01 | 选模板+选方案 | 从已有模板列表选一个，关联新上传的方案 |
| C02 | 一键生成 | 知识库字段自动映射到模板占位符，生成 .docx |
| C03 | 填充率报告 | 展示自动填充/需确认/人工填写 的统计 |
| C04 | 下载 | 下载 .docx 格式 |
| C05 | 批量生成 | 一个方案同时生成多份文档（DMC/DMP/SAP） |

### 模块D：客户管理（P2 · 增值）

| ID | 功能 | 描述 |
|---|---|---|
| D01 | 客户列表 | 管理已合作的申办方信息 |
| D02 | 客户-模板标签 | 给模板打客户标签，方便筛选 |

## 5. MVP范围（Phase 1）

| 模块 | 包含 | 暂不包含 |
|---|---|---|
| 模板管理 | 上传+解析+确认+列表 | 版本管理+标签 |
| 知识库 | **批量入库+单方案提取+校验** | 知识提炼闭环+检索 |
| 文档生成 | 选模板+方案→一键生成DMC→下载 | 批量生成+填充率 |

**Phase 1 核心验证问题：**
1. 用户能否批量导入历史PDF并成功构建知识库？
2. 新方案上传后能否走通「提取→校验→生成→下载」的完整链路？
3. 生成的DMC Charter质量是否达到可用的水平？
